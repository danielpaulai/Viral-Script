# IMPLEMENTATION 1: FIX SCRIPT OUTPUT QUALITY

## Problem
Scripts are generating "random fluff" instead of actionable, valuable content.

## Solution
Add a **Quality Gate** that grades scripts and regenerates if below threshold.

### Add This to the System Prompt:

```
<output_quality_rules>
EVERY SCRIPT MUST:

1. BE ACTIONABLE
- Every script must give the viewer ONE clear thing they can do immediately after watching
- No vague advice like "think about your goals" or "consider your options"
- Instead: Specific steps like "Open your phone. Go to Settings. Turn off notifications for 2 hours."

2. HAVE CONCRETE SPECIFICS
- Include specific numbers, timeframes, or examples
- BAD: "Post more content"
- GOOD: "Post 3 times per day for 30 days"

3. AVOID FLUFF PHRASES
Never use these fluffy phrases:
- "In today's world..."
- "Have you ever wondered..."
- "Let me tell you something..."
- "The truth is..."
- "Here's the thing..."
- "At the end of the day..."
- "It's important to remember..."
- "What most people don't realize..."

4. GET TO THE POINT
- Hook in first 2 seconds (first sentence)
- Main value by second 3-5
- No long intros or build-ups
- Every sentence must earn its place

5. END WITH CLEAR NEXT STEP
- Tell them exactly what to do next
- Make it doable in under 5 minutes
- Be specific: "Comment 'GUIDE' below" not "Let me know what you think"

</output_quality_rules>

<quality_check>
Before outputting any script, internally grade it 1-10 on:
- Actionability (Can viewer do something specific after watching?)
- Specificity (Are there concrete numbers, steps, examples?)
- Fluff-free (No wasted sentences or vague phrases?)
- Hook strength (Does first sentence demand attention?)
- Clear CTA (Is the next step obvious?)

If total score is below 35/50, rewrite the script before showing to user.

DO NOT show this grading to the user. Just ensure the script meets the bar.
</quality_check>
```

### Example of BAD vs GOOD Output:

**BAD OUTPUT (Current Problem):**
```
Have you ever thought about what makes content go viral?

The truth is, it's not as complicated as you might think.

In today's world, everyone is fighting for attention.

But here's the thing - the best creators know something others don't.

They understand that value is key.

So next time you create content, think about what value you're providing.

Let me know your thoughts in the comments!
```
**Problems:** Fluffy intro, vague advice, no specifics, weak CTA

**GOOD OUTPUT (What We Want):**
```
I analyzed 500 viral videos last month.

One pattern showed up in 89% of them.

The hook answered a question the viewer was already asking themselves.

Not "here's how to grow" - that's what YOU want to talk about.

But "why your last 10 posts flopped" - that's what THEY'RE thinking.

Here's the formula:

[Pain they feel] + [Unexpected reason] + [Quick fix]

Example: "Your posts are flopping because you're posting at 9am. Here's the 3-minute fix."

Try this on your next post.

Screenshot your analytics before and after.

Drop "RESULTS" in the comments in 7 days - I want to see your wins.
```
**Why it works:** Specific numbers (500, 89%), clear formula, actionable step, specific CTA with timeframe

# IMPLEMENTATION 2: DEEP RESEARCH MODE FOR TOPIC EXPANSION

## Problem
Users paste a raw/vague idea and get a mediocre script because the AI doesn't have enough context.

## Solution
Add a "Deep Research Mode" toggle that expands a raw idea into a detailed brief BEFORE generating the script.

│  │ Turn ON if your idea is raw/basic.                        │  │
│  │ AI will expand it into a detailed brief before            │  │
│  │ generating your script.                                   │  │
│  └───────────────────────────────────────────────────────────┘  │

### Deep Research Prompt (Add to Backend):

```
<deep_research_mode>
When Deep Research Mode is enabled, BEFORE generating a script:

STEP 1: EXPAND THE RAW IDEA
Take the user's raw idea and create a detailed brief by answering:

1. CORE MESSAGE
- What is the ONE main point of this video?
- What should the viewer remember after watching?

2. TARGET VIEWER
- Who specifically needs to hear this?
- What problem are they facing right now?
- What have they already tried that didn't work?

3. UNIQUE ANGLE
- What's the unexpected twist on this topic?
- Why is the common advice wrong?
- What insider knowledge can we share?

4. PROOF/CREDIBILITY
- What numbers, stats, or examples support this?
- What personal experience makes this credible?
- What results has this approach gotten?

5. ACTIONABLE TAKEAWAY
- What's the ONE thing they can do in the next 5 minutes?
- How will they know it worked?

STEP 2: SHOW THE EXPANDED BRIEF
Display this to the user in a clean format:

---
📋 EXPANDED VIDEO BRIEF

**Core Message:** [One sentence]

**Target Viewer:** [Who this is for and their current struggle]

**Unique Angle:** [The unexpected twist]

**Key Proof Points:**
- [Stat/example 1]
- [Stat/example 2]

**Actionable Takeaway:** [Specific next step]

STEP 3: ASK FOR CONFIRMATION
"Does this capture what you want to say? 
[✓ Yes, generate my script] [✏️ Let me edit this first]"

Only generate the script AFTER user confirms the brief.
</deep_research_mode>
```

# IMPLEMENTATION 3: GENERATE 3 SCRIPT VERSIONS WITH RANKED HOOKS

## Problem
Users don't know which hook is best for their topic. Currently they pick randomly from 50 options.

## Solution
Auto-select the top 3 best hooks for their topic and generate 3 script versions (ranked).

### How It Works:

```
<hook_matching_system>

STEP 1: ANALYZE THE TOPIC
When user submits a topic, categorize it:
- Is it a PERSONAL STORY? → Use hooks #1-8 (Personal Experience)
- Is it a CASE STUDY about someone/something? → Use hooks #9-16 (Case Study)
- Is it revealing something HIDDEN/SECRET? → Use hooks #17-26 (Secret Reveal)
- Is it a CONTRARIAN take? → Use hooks #27-34 (Contrarian)
- Is it answering a QUESTION people have? → Use hooks #35-42 (Question)
- Is it a LIST of things? → Use hooks #43-47 (Listicle)
- Is it about a COMMON MISTAKE? → Use hooks #48-50 (Mistake)

STEP 2: SELECT TOP 3 HOOKS
Based on the categorization, select the 3 most relevant hooks.
Consider:
- Topic fit (does the hook template match the content?)
- Engagement potential (which hooks drive most curiosity?)
- Uniqueness (avoid hooks that are overused for this topic)

STEP 3: GENERATE 3 VERSIONS
Create 3 complete scripts, each using a different hook.
Rank them:
- 🥇 VERSION A (Recommended) - Best hook match
- 🥈 VERSION B (Alternative) - Strong second option  
- 🥉 VERSION C (Creative) - Unexpected angle

STEP 4: DISPLAY TO USER
Show all 3 versions with:
- The hook used
- Why it was selected
- The full script
- One-click copy

</hook_matching_system>
```

# IMPLEMENTATION 4: SIMPLIFY VIDEO TYPE OPTIONS

## Current Problem
Too many video type options that most users won't use.

## Solution
Reduce to 3 simple options (for now):

```
Video Type:
○ 🎤 Talking Head (You speak to camera)
○ 🎬 B-Roll + Voiceover (Voice over footage, no face)
○ 📝 Text on Screen (No voice, just text cards)
```

### Remove These (For Now):
- ❌ AI Avatar / Clone Video
- ❌ Screen Recording / Tutorial
- ❌ Mixed Format

### Why:
> "Majority of them will do probably either a B-roll or talking head. They probably might not have a screen recording or a mixed format."
